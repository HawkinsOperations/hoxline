# Hoxline Blueprint

Hoxline is the HawkinsOperations ProofOps control plane for AI-assisted security work.

AevumGuard was a prior working name. Hoxline is the current product name.

Hoxline governs how AI-assisted security work becomes tested, reviewed, blocked, or safe to claim.

Doctrine: AI is not the authority. Evidence is.

## Core Loop

AI-assisted security work
→ Artifact Intake
→ Evidence Graph
→ Telemetry Contract Check
→ Controlled Validation
→ Runtime Candidate Ledger
→ Signal Observation
→ Human Review Gate
→ ProofCard
→ Claim Authority
→ Safe Claim / Blocked Claim

## Product Role

Hoxline is the control plane that records state, gates promotion, and constrains claims. It does not treat AI output as authority. Every promoted claim must be traceable to evidence, validation state, review state, and Claim Authority disposition.

## v0 Spine

The v0 product spine contains:

* Product boundary and doctrine.
* Module map for every loop stage.
* Seven-repo system map.
* Gauntlet walkthrough for one artifact.
* Promotion state schema.
* Evidence graph schema.
* Sample gauntlet records.

## Internal Capabilities

Claim Firewall is the first Claim Authority enforcement capability inside Hoxline. It evaluates whether proposed claim wording is allowed by the available evidence state.

Claim Firewall is not the product, not the platform, not the front-door repository, and not the whole control plane.

## Proof Ceiling

The blueprint defines structure only. It does not claim runtime observation, signal observation, external safety status, final authorization, or deployment status.
